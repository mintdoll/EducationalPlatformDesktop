Введение
Современные образовательные организации, особенно в сфере дополнительного профессионального образования, нуждаются в автоматизации основных процессов обучения и управления контентом. Одной из таких сфер является деятельность онлайн-школ, в которых ежедневно выполняются операции по авторизации слушателей, записи на курсы, предоставлению доступа к учебным материалам, проведению тестирований, выдаче сертификатов и контролю успеваемости. Во многих организациях подобные процессы до сих пор ведутся вручную либо с использованием разрозненных документов и электронных таблиц, что приводит к снижению скорости обработки информации, появлению ошибок и затрудняет контроль образовательной деятельности.
Актуальность выбранной темы заключается в необходимости повышения эффективности работы онлайн-школы за счёт внедрения программного обеспечения, обеспечивающего централизованное хранение данных, ускорение обслуживания слушателей, снижение количества ошибок при оформлении образовательных заявок и улучшение управляемости учебным процессом.
В ходе выполнения работы было разработано настольное приложение для онлайн-школы дополнительного профессионального образования, взаимодействующее с веб-сервисом через унифицированный API. Приложение реализовано на языке C# с использованием технологии Windows Presentation Foundation (WPF), что позволило создать удобный, функциональный и визуально привлекательный пользовательский интерфейс с поддержкой современной архитектуры MVVM. Разработанный десктоп-клиент обеспечивает пользователям полный доступ ко всем функциям платформы: авторизацию, выбор курсов, изучение материалов, прохождение тестов и получение результатов обучения. Наличие настольного приложения расширяет способы взаимодействия с системой, обеспечивает стабильную работу в условиях нестабильного интернет-соединения за счёт локального кэширования данных и повышает общее удобство использования образовательной платформы.
Исходный код проекта опубликован в открытом репозитории на GitHub: [https://github.com/mintdoll/EducationalPlatformDesktop.git].

Выбранный стек технологий
Visual Studio / Visual Studio Code – это современные среды разработки программного обеспечения, предназначенные для написания, отладки и тестирования исходного кода. Они поддерживают работу с различными языками программирования и технологиями, а также позволяют подключать дополнительные расширения для повышения эффективности разработки. В проекте среда разработки используется для создания серверной части веб-сервиса, клиентской части приложения и настольного WPF-приложения.
Git – это распределённая система контроля версий, предназначенная для управления изменениями исходного кода. Она позволяет отслеживать историю разработки, создавать отдельные ветки проекта и обеспечивать совместную работу нескольких разработчиков. В проекте Git используется для хранения исходного кода программного комплекса и контроля изменений в процессе разработки.
GitHub – это веб-платформа для хранения и управления Git-репозиториями. Она предоставляет инструменты для совместной разработки, резервного хранения исходного кода и управления версиями проекта. В проекте GitHub применяется для размещения исходного кода настольного приложения.
ASP.NET Core – кроссплатформенный фреймворк, на котором разработана серверная часть программного комплекса. Настольное приложение взаимодействует с этим бэкендом через единый REST API для авторизации, получения данных о курсах и передачи результатов тестирования.
PostgreSQL – истема управления базами данных, используемая в серверной части комплекса. Настольное приложение обращается к данным через API, не взаимодействуя с СУБД напрямую.
C# – это современный объектно-ориентированный язык программирования платформы .NET. Он используется для разработки серверных приложений, бизнес-логики и клиентских программ. В проекте C# применяется при создании серверной приложения.
WPF (Windows Presentation Foundation) – это технология разработки графических интерфейсов приложений под операционную систему Windows. Она предоставляет инструменты для создания оконных приложений с современным пользовательским интерфейсом. В проекте WPF используется для разработки настольного приложения студента, обеспечивающего авторизацию, просмотр курсов, работу с учебными материалами и прохождение тестирования.
JWT (JSON Web Token) – это стандарт передачи данных для безопасной аутентификации пользователей. Он позволяет создавать защищённые сессии и контролировать доступ пользователей к функциям системы. В проекте JWT используется для авторизации пользователей веб-сервиса и настольного приложения.
Figma – это онлайн-инструмент для проектирования пользовательских интерфейсов и создания прототипов приложений. Он позволяет разрабатывать макеты экранов и визуализировать структуру будущей системы. Введение
Наименование системы
Полное наименование системы
Настольное приложение для онлайн-школы дополнительного образования.
Условное обозначение темы разработки
Разработка настольного приложения.
Краткая характеристика области применения программы.
Программа предназначена для обеспечения студентам онлайн-школы удобного доступа к образовательным курсам посредством настольного графического приложения. Приложение реализует функции авторизации, просмотра профиля и списка доступных курсов, навигации по модулям и урокам, чтения текстовых лекций, прохождения тестов, отслеживания прогресса обучения и просмотра сведений о полученных сертификатах.
Настольное приложение разработано на базе WPF (.NET / C#) и функционирует как самостоятельный графический клиент, взаимодействующий с серверной частью образовательной платформы через выделенный REST API. Комплекс может быть использован в образовательных учреждениях, корпоративных учебных центрах и частными лицами для доступа к дистанционному обучению.
Техническое задание разработано в соответствии с ГОСТ 19.201-78. Настоящее техническое задание распространяется на разработку настольного приложения для студентов онлайн-платформы обучения физических лиц и организаций по направлениям: IT, бухгалтерия, экономика, логистика, закупки.

Назначение разработки
Функциональное назначение
Платформа включает:
авторизацию пользователя (ввод email/MAX и пароля, получение JWT-токена сессии);
просмотр профиля и списка доступных и купленных курсов в графическом интерфейсе;
навигацию по модулям и урокам выбранного курса с отображением текстового контента лекций в окне приложения;
прохождение тестов: поочередный вывод вопросов (с одним или несколькими вариантами ответов), интерактивный ввод ответов, отправка результатов на сервер;
просмотр текущего прогресса обучения;
просмотр информации о полученных сертификатах (уникальный номер).
Эксплуатационное назначение
Настольное приложение предназначено для эксплуатации в режиме клиент-сервер. Круг пользователей: студенты онлайн-школы. Условия эксплуатации: периодическое подключение к серверу (к API серверной части программного комплекса) для получения данных и передачи результатов тестирования.

Настольное приложение должно иметь следующие элементы:
окно авторизации;
главное окно;
карточка курса;
страница курса;
пройденные уроки;
прохождение тестов;
результаты тестирования;
раздел прогресса;
раздел сертификатов;
кнопка выхода из системы.
Авторизация
Функции:
авторизация пользователя (ввод email/логина и пароля, получение JWT-токена сессии);
проверка корректности учетных данных при входе в систему;
завершение текущей сессии пользователя (выход из системы);
защита от несанкционированного доступа при неверном вводе учетных данных.
 Главное окно и навигация
Обязательные функции:
отображение информации о пользователе (ФИО);
навигационная панель для перехода между разделами;
переход к списку курсов;
переход к просмотру прогресса обучения;
переход к списку сертификатов;
кнопка выхода из системы.
Просмотр курсов
Функции:
просмотр списка доступных и приобретённых курсов;
отображение карточки каждого курса с указанием названия, краткого описания и процента прогресса;
визуальное отображение статуса курса (не начат, в процессе, завершён);
переход к выбранному курсу для начала или продолжения обучения.
Функционал курса
Студент должен иметь возможность:
просматривать структуры курса в виде дерева модулей и уроков;
выбора урока для изучения;
отображение текстового контента лекции в основной рабочей области окна;
видеть прогресс.
Тестирование
Приложение должно поддерживать:
поочередный вывод вопросов с одним или несколькими вариантами ответов;
интерактивный ввод ответов с клавиатуры;
отправку результатов тестирования на сервер через API в реальном времени;
отображение результата пользователю.
Сертификаты
Сертификат выдается автоматически при условиях:
курс пройден минимум на 90%;
итоговый тест сдан.
Сертификат должен содержать:
ФИО студента;
название курса;
дату выдачи;
уникальный номер.
Требования к надёжности
Система сохраняет работоспособность при некорректных действиях пользователя.
Условия эксплуатации
Работа приложения предполагает периодическое подключение к серверному API. Допустима работа с локально развёрнутым сервером.
Требования к составу и параметрам технических средств
Для разработки и тестирования настольного приложения достаточно персонального компьютера под управлением Windows с процессором не менее 2 ядер (Intel Core i3 или AMD Ryzen 3), оперативной памятью не менее 4 ГБ и свободным дисковым пространством не менее 10 ГБ.
Для работы пользователя необходим компьютер под управлением Windows 10 или выше с установленной средой выполнения .NET актуальной LTS-версии и доступом к серверному API (локально или через сеть Интернет).
Требования к информационной и программной совместимости
Настольное приложение разрабатывается на платформе .NET с использованием технологии WPF (C#). Авторизация осуществляется через получение JWT-токена от серверного API. Взаимодействие с системой реализовано через RESTful API серверной части комплекса. Код должен быть структурирован по принципу разделения ответственности между представлением (XAML/code-behind), бизнес-логикой и сервисами доступа к API.
Требования к архитектуре программного продукта
Настольное приложение проектируется как независимый WPF-клиент, взаимодействующий с бэкенд-ядром исключительно через RESTful API. Приложение не обращается к базе данных напрямую. 
Компоненты приложения:
модуль авторизации: получение JWT-токена через API;
модуль просмотра курсов: запросы к API для получения списка курсов, модулей, уроков и текстового контента;
модуль тестирования: получение вопросов теста через API, отправка результатов;
модуль прогресса и сертификатов: получение данных о прогрессе и сертификатах через API.

Настоящий документ содержит эскизный проект настольного приложения для онлайн-школы дополнительного образования.
Эскизный проект разработан на стадии проектирования автоматизированной системы и содержит предварительные проектные решения по составу, структуре, функциям, информационному обеспечению, программному обеспечению, техническому обеспечению, организационному обеспечению и пользовательскому интерфейсу системы.

Вид системы
Настольное приложение является автоматизированной информационной системой, предназначенной для обеспечения студентам онлайн-школы доступа к образовательным курсам, учебным материалам, системе тестирования и сведениям о результатах обучения посредством нативного графического интерфейса под управлением Windows.
Назначение системы
Настольное приложение предназначено для автоматизации процесса дистанционного обучения студентов онлайн-школы и позволяет осуществлять следующие функции:
авторизация пользователя по логину (email / MAX) и паролю с получением JWT-токена;
просмотр данных профиля авторизованного студента;
просмотр списка доступных и купленных курсов;
навигация по модулям и урокам выбранного курса;
чтение текстовых лекций в окне приложения;
прохождение тестов: поочерёдный вывод вопросов с вариантами ответов, ввод ответов, отправка результатов на сервер через API;
просмотр текущего прогресса обучения;
просмотр сведений о полученных сертификатах.
Макет настольного приложения
Дизайн настольного приложения выполнен в сдержанном стиле для снижения нагрузки на глаза при длительной работе с учебными материалами и обеспечения быстрой адаптации пользователей.

Проектирование взаимодействия настольного приложения с серверным API
Настольное приложение образовательной платформы разработано по клиент-серверной архитектуре и не осуществляет прямого взаимодействия с базой данных. Получение, изменение и сохранение данных выполняется через серверную часть программного комплекса посредством REST API.
Серверная часть реализована на платформе ASP.NET Core Web API и выступает единым центром обработки данных. Настольное приложение, разработанное с использованием технологии WPF, обращается к серверу посредством HTTP-запросов и получает данные в формате JSON.
Такой подход обеспечивает независимость клиентского приложения от структуры базы данных и позволяет изменять серверную реализацию без необходимости модификации клиентской части.
Архитектура взаимодействия компонентов представлена следующим образом:
Пользователь → WPF-приложение → REST API → Сервер приложений → База данных PostgreSQL.
В процессе работы настольное приложение выполняет следующие основные операции взаимодействия с API:
авторизация пользователя;
получение данных профиля;
получение списка доступных курсов;
получение информации о модулях и уроках;
получение содержимого учебных материалов;
получение тестовых заданий;
отправка результатов тестирования;
получение данных о прогрессе обучения;
получение информации о сертификатах.
Для передачи данных используется формат JSON, обеспечивающий компактность хранения информации и удобство обработки на стороне клиента и сервера.
Авторизация пользователя
При запуске настольного приложения пользователю отображается окно авторизации. Для получения доступа к функциональным возможностям системы пользователь должен ввести логин и пароль своей учетной записи.
После ввода учетных данных настольное приложение формирует запрос к серверному API, который выполняет проверку корректности введенных данных и определяет роль пользователя в системе.
В случае успешной проверки пользователю предоставляется доступ к функциональным модулям приложения в соответствии с назначенными правами доступа.
При вводе некорректного логина или пароля сервер возвращает сообщение об ошибке авторизации, после чего пользователю предлагается повторно выполнить вход в систему.
Получение информации о курсах
После успешного входа в систему приложение получает список курсов, доступных пользователю.
Для каждого курса отображаются:
название курса;
краткое описание;
процент прохождения;
текущий статус обучения.
Полученные данные отображаются в виде карточек курсов в пользовательском интерфейсе.
Получение структуры курса
При выборе курса приложение запрашивает подробную информацию о программе обучения.
Сервер передает:

список уроков;
сведения о преподавателе;
продолжительность обучения;
текущий прогресс пользователя.
Полученная информация отображается в виде древовидной структуры курса.
Получение учебных материалов
При открытии урока приложение отправляет запрос к серверному API для получения содержимого учебного материала.
В ответ сервер возвращает:
название урока;
текст лекции
дополнительные материалы;
сведения о прохождении урока.
Учебный материал отображается в основной рабочей области приложения.
Взаимодействие с модулем тестирования
После завершения изучения учебного материала пользователь может приступить к тестированию.
Настольное приложение получает от сервера:

правила прохождения теста.
После завершения тестирования ответы пользователя отправляются на сервер для автоматической проверки.
Результаты проверки возвращаются клиентскому приложению и отображаются пользователю.
Получение данных о прогрессе обучения
Для контроля процесса обучения приложение периодически получает сведения о прогрессе пользователя.
Передаваемые данные включают:
количество завершённых уроков;
количество пройденных тестов;
процент прохождения курса;
текущий статус обучения.
Полученная информация используется для формирования визуального отображения прогресса обучения.
Получение информации о сертификатах
После успешного завершения курса приложение получает сведения о выданных сертификатах.
Для каждого сертификата отображаются:
название курса;

дата выдачи;

Информация отображается в отдельном разделе пользовательского интерфейса.
Организация обмена данными
Все запросы между клиентским приложением и сервером передаются по протоколу HTTPS.
Для выполнения сетевых запросов используется встроенный класс HttpClient платформы .NET.
Обработка полученных данных выполняется специализированными сервисами приложения, реализованными в соответствии с архитектурным шаблоном MVVM.
Использование REST API обеспечивает слабую связанность компонентов программного комплекса, упрощает сопровождение системы и позволяет использовать единый сервер для работы различных клиентских приложений.
Проектирование интерфейсов
Диаграмма классов, отражающая основные компоненты настольного приложения: модели данных, сервисы взаимодействия с API, сервис авторизации и модели представления (ViewModel) архитектурного шаблона MVVM представлена на рисунке А.3 приложения А.
Обозначение символов:
«+» – открытый (public) член класса, доступный из любой части приложения;
«−» – закрытый (private) член класса, доступный только внутри класса;
стрелка с пунктиром – отношение зависимости: один класс использует другой при выполнении своих функций.
Модели представления (ViewModel) – LoginViewModel, CoursesViewModel, LessonViewModel, TestViewModel – обеспечивают связь между пользовательским интерфейсом и данными приложения. Каждая модель представления зависит от сервиса ApiService, который выполняет HTTP-запросы к серверному API и преобразует ответы в объекты моделей данных.
Модуль авторизации использует AuthService, отвечающий за выполнение входа в систему, получение JWT-токена и управление пользовательской сессией. После успешной авторизации токен передаётся в заголовке Authorization при всех последующих запросах к защищённым ресурсам API.
Модели данных – Course, Lesson, Certificate – соответствуют структуре объектов, передаваемых через REST API, и используются для хранения данных, полученных от сервера, и их отображения в пользовательском интерфейсе.

Описание основных модулей программы
Модуль авторизации
Модуль авторизации обеспечивает вход пользователя в систему. Пользователь вводит email и пароль, после чего WPF-клиент отправляет данные на серверный API. При успешной проверке сервер возвращает JWT-токен, который сохраняется в текущей сессии и используется при дальнейших защищенных запросах.
Логика авторизации реализована в файле ViewModels/LoginViewModel.cs. Перед отправкой запроса приложение проверяет, что поля email и пароля заполнены. При ошибке подключения, неверных данных или превышении времени ожидания пользователю выводится соответствующее сообщение.
Выборочный фрагмент логики авторизации:
var email = Login.Trim();
if (string.IsNullOrWhiteSpace(email) ||
    string.IsNullOrWhiteSpace(Password))
{
    ErrorMessage = "Введите email и пароль.";
    SuccessMessage = string.Empty;
    return;
}
AppSession.Clear();
_apiClient.SetBearerToken(null);
var response = await _apiClient.LoginAsync(new LoginRequestDto
{
    Email = email,
    Password = Password
});
if (response == null ||
    string.IsNullOrWhiteSpace(response.Token))
{
    ErrorMessage = "Неверный email или пароль.";
    return;
}
После успешного входа данные пользователя сохраняются в AppSession, а полученный токен передается в API-клиент:
AppSession.UserId = response.UserId;
AppSession.Role = response.Role;
AppSession.Email = response.Email ?? email;
AppSession.Token = response.Token;
AppSession.Name = response.Name;
AppSession.LastName = response.LastName;
_apiClient.SetBearerToken(response.Token);
SuccessMessage = "Вход выполнен успешно.";
LoginSucceeded?.Invoke();
После успешной авторизации пользователь переходит к главному окну приложения. При ошибке авторизации отображается сообщение о неверно введенных учетных данных или о невозможности подключения к серверу.
Модуль взаимодействия с API
Для обмена данными с сервером используется отдельный сервис OnlineSchoolApiClient, расположенный в файле Api/Services/OnlineSchoolApiClient.cs. Он выполняет HTTP-запросы, получает ответы в формате JSON и преобразует их в объекты DTO.
При создании API-клиента задается базовый адрес сервера, а после авторизации в заголовки запросов добавляется Bearer-токен.
Выборочный фрагмент установки токена:
public void SetBearerToken(string? token)
{
    if (string.IsNullOrWhiteSpace(token))
    {
        _httpClient.DefaultRequestHeaders.Authorization = null;
        return;
    }
    _httpClient.DefaultRequestHeaders.Authorization =
        new AuthenticationHeaderValue("Bearer", token);
}
Через данный сервис WPF-клиент выполняет следующие основные операции:
авторизация пользователя;
получение списка курсов;
получение курса по идентификатору;
получение уроков выбранного курса;
получение тестов по курсу;
отправка результата завершения теста;
получение прогресса пользователя;
получение сертификата.
Выборочный фрагмент получения списка курсов:
public async Task<List<CourseDto>> GetCoursesAsync(
    CancellationToken cancellationToken = default)
{
    var courses = await _httpClient.GetFromJsonAsync<List<CourseDto>>(
        "api/Course",
        JsonOptions,
        cancellationToken);
    return courses ?? new List<CourseDto>();
}
Выборочный фрагмент получения уроков курса:
public async Task<List<LessonDto>> GetLessonsByCourseIdAsync(
    int courseId,
    CancellationToken cancellationToken = default)
{
    var lessons = await _httpClient.GetFromJsonAsync<List<LessonDto>>(
        $"api/Lesson/course/{courseId}",
        JsonOptions,
        cancellationToken);
    return lessons ?? new List<LessonDto>();
}
Такой подход позволяет отделить сетевую логику от интерфейса приложения и упростить сопровождение программы. Пользовательский интерфейс работает с данными через модели представления, а все HTTP-запросы сосредоточены в одном сервисе.
Модуль курсов и уроков
Модуль курсов предназначен для отображения списка доступных студенту курсов. Каждый курс содержит название, направление, преподавателя, описание, признак приобретения, проходной балл, список модулей и уроков, а также сведения о прогрессе прохождения.
Основная логика работы с курсами и уроками находится в MainViewModel.cs. При выборе курса приложение обновляет список модулей, а при выборе модуля - список уроков. После открытия урока его содержимое отображается в рабочей области приложения.
Выборочный фрагмент обновления модулей выбранного курса:
private void UpdateModulesForSelectedCourse()
{
    Modules.Clear();
    Lessons.Clear();
    _selectedModule = null;
    _selectedLesson = null;
    if (SelectedCourse == null)
        return;
    foreach (var module in SelectedCourse.Modules)
        Modules.Add(module);
    if (Modules.Count > 0)
        SelectedModule = Modules.First();
}
Выборочный фрагмент открытия урока:
private void OpenLesson(Lesson? lesson)
{
    if (lesson == null)
        return;
    SelectedLesson = lesson;
    LessonContent = lesson.Content;
    CurrentView = _lessonView;
    PageTitle = lesson.Title;
    PageDescription = SelectedCourse != null
        ? $"{SelectedCourse.Title} · {SelectedModule?.Title}"
        : "Текст лекции";
}
Модель Course рассчитывает процент прохождения курса на основе количества завершенных уроков:
public int LessonProgressPercent
{
    get
    {
        if (TotalLessons == 0)
        {
            return 0;
        }
        return (int)Math.Round(
            (double)CompletedLessons / TotalLessons * 100);
    }
}
Модель Lesson хранит статус прохождения урока. После отметки урока как пройденного интерфейс автоматически обновляет текст кнопки и статус урока:
public string CompletionButtonText => IsCompleted
    ? "Урок пройден"
    : "Отметить пройденным";
public string CompletionStatus => IsCompleted
    ? "✓ Пройден"
    : "Не пройден";
Модуль тестирования
Модуль тестирования обеспечивает отображение вопросов, выбор вариантов ответа, подсчет правильных ответов, расчет процента выполнения и формирование результата тестирования. Логика теста реализована в ViewModels/TestViewModel.cs, а пользовательский интерфейс - в Views/TestView.xaml.
Пользователь может выбирать ответы мышью или клавишами 1-4, а переходить к следующему вопросу клавишей Enter.
Выборочный фрагмент привязки клавиш в TestView.xaml:
<KeyBinding Key="D1"
            Command="{Binding SelectOptionCommand}"
            CommandParameter="0" />
<KeyBinding Key="D2"
            Command="{Binding SelectOptionCommand}"
            CommandParameter="1" />
<KeyBinding Key="D3"
            Command="{Binding SelectOptionCommand}"
            CommandParameter="2" />
<KeyBinding Key="D4"
            Command="{Binding SelectOptionCommand}"
            CommandParameter="3" />
<KeyBinding Key="Enter"
            Command="{Binding AnswerCommand}" />
Выборочный фрагмент обработки ответа:
private void Answer()
{
    var selectedAnswer =
        CurrentQuestion.Options[SelectedOption];
    if (selectedAnswer.IsCorrect)
    {
        _correctAnswers++;
    }
    if (_currentIndex < TotalQuestions - 1)
    {
        _currentIndex++;
        SelectedOption = -1;
        OnPropertyChanged(nameof(CurrentQuestion));
        OnPropertyChanged(nameof(QuestionNumber));
        OnPropertyChanged(nameof(QuestionProgress));
        return;
    }
    IsFinished = true;
}
После завершения теста формируется объект TestResult, содержащий номер попытки, идентификатор курса, идентификатор теста, количество правильных ответов, общее количество вопросов, процент выполнения, проходной балл и дату завершения.
Выборочный фрагмент формирования результата:
var result = new TestResult
{
    AttemptId = Guid.NewGuid().ToString("N"),
    CourseId = _test.CourseId,
    TestId = _test.Id,
    TestTitle = _test.Title,
    CorrectAnswers = CorrectAnswers,
    TotalQuestions = TotalQuestions,
    ScorePercent = ScorePercent,
    PassingScore = _test.PassingScore,
    IsPassed = IsPassed,
    CompletedAt = DateTime.Now
};
TestCompleted?.Invoke(result);
После завершения теста WPF-клиент отображает результат проверки, количество правильных ответов и статус прохождения. Дополнительно приложение отправляет результат на сервер через метод FinishTestAsync. Если сервер недоступен, используется локальный результат тестирования.
Модуль профиля, прогресса и сертификатов
Модуль профиля отображает сведения об авторизованном пользователе, сохраненные в текущей сессии: идентификатор пользователя, роль, email, имя и фамилию. Данные сессии хранятся в классе AppSession.
Выборочный фрагмент хранения данных пользователя:

Модуль прогресса показывает средний процент обучения, количество завершенных курсов, количество пройденных уроков и результат тестирования по каждому курсу. Экран прогресса реализован в Views/ProgressView.xaml, а обновление данных выполняется в MainViewModel.
Выборочный фрагмент обновления прогресса курса:
private void UpdateProgressForCourse(Course course)
{
    var oldItem = ProgressItems.FirstOrDefault(
        item => item.CourseId == course.Id);
    var newItem = CreateProgressItem(course);
    if (oldItem == null)
    {
        ProgressItems.Add(newItem);
        return;
    }
    var index = ProgressItems.IndexOf(oldItem);
    ProgressItems[index] = newItem;
}
Модуль сертификатов выводит сведения о полученных сертификатах: номер сертификата, название курса, ФИО студента, дату выдачи и итоговый результат. Модель сертификата описана в файле Models/Certificate.cs.
Выборочный фрагмент модели сертификата:
public class Certificate
{
    public string Id { get; set; } = string.Empty;
    public string Number { get; set; } = string.Empty;
    public int CourseId { get; set; }
    public string CourseName { get; set; } = string.Empty;
    public string StudentName { get; set; } = string.Empty;
    public DateTime IssuedAt { get; set; }
    public int Score { get; set; }
    public string IssuedDate =>
        IssuedAt.ToString("dd.MM.yyyy");
    public string ScoreText =>
        $"Итоговый результат: {Score}%";
}
Сертификат создается только после выполнения условий завершения курса: студент должен пройти не менее 90% уроков и успешно сдать итоговый тест.
Выборочный фрагмент проверки условий выдачи сертификата:
if (!course.CanReceiveCertificate ||
    !result.IsPassed)
{
    return;
}
var alreadyExists = Certificates.Any(
    certificate => certificate.CourseId == course.Id);
if (alreadyExists)
{
    return;
}
Для сохранения демонстрационного состояния используется сервис DemoStateStorage. Он записывает прогресс, пройденные уроки, лучший результат теста и сертификаты в файл demo-state.json в папке данных пользователя Windows. Это позволяет сохранять состояние приложения между запусками.

Общие сведения тестирования
Наименование системы
Настольное приложение для онлайн-школы дополнительного образования.
Назначение системы
WPF-клиент предназначен для предоставления студенту доступа к функциям онлайн-школы: авторизации, просмотру курсов, изучению учебных материалов, прохождению тестов, просмотру прогресса и сертификатов.
Цель испытаний
Целью испытаний является проверка соответствия WPF-клиента требованиям технического задания, а также проверка корректности работы интерфейса, авторизации, взаимодействия с REST API, загрузки данных, прохождения тестирования и отображения сообщений об ошибках.

Руководство предназначено для конечного пользователя WPF-клиента - студента онлайн-школы. Для работы с приложением пользователю не требуется специальная техническая подготовка.
Документ разработан в соответствии с ГОСТ 19.101-77.

Назначение и применение системы
Назначение системы
Настольное приложение предназначено для работы студента с образовательной платформой онлайн-школы дополнительного образования.
Приложение обеспечивает:
авторизацию студента;
просмотр данных профиля;
просмотр списка доступных курсов;
просмотр описания курса;
переход по модулям и урокам;
изучение текстовых учебных материалов;
прохождение тестирования;
просмотр результатов тестов;
просмотр прогресса обучения;
просмотр сведений о полученных сертификатах;
завершение пользовательской сессии.
Пользователи системы
В WPF-клиенте предусмотрена одна основная роль пользователя – студент.
Студент получает доступ к образовательным материалам, тестам, результатам обучения и сертификатам после успешной авторизации.
Применение системы
Для работы с приложением необходимо:
персональный компьютер или ноутбук с операционной системой Windows;
установленное настольное приложение онлайн-школы;
доступ к сети Интернет;
доступность серверной части приложения;
логин и пароль пользователя;
наличие учетной записи студента.
Взаимодействие с системой
Для начала работы необходимо запустить настольное WPF-приложение. После запуска открывается окно авторизации.
Авторизация в приложение
Для входа в систему необходимо.
Ввести логин.
Ввести пароль.
Нажать кнопку «Войти».
После успешной авторизации откроется главное окно приложения. Если логин или пароль введены неверно, приложение отобразит сообщение об ошибке, после чего пользователь сможет повторить попытку входа.
Общие принципы
Интерфейс системы состоит из следующих элементов:
окно авторизации;
главное окно приложения;
навигационная панель;
основная рабочая область;
карточки курсов;
экран просмотра урока;
экран прохождения теста;
раздел профиля;
раздел сертификатов.
Для перехода между разделами пользователь выбирает нужный пункт меню. Содержимое выбранного раздела отображается в основной рабочей области приложения.
Для завершения работы необходимо нажать кнопку «Выйти». После этого текущая сессия завершается, и пользователь возвращается на экран авторизации.
Действия пользователей
Просмотр профиля
Для просмотра профиля необходимо перейти в соответствующий раздел приложения. На экране отображаются сведения об авторизованном пользователе: ФИО, логин, группа и роль.
Просмотр курсов
Для просмотра курсов необходимо открыть раздел «Курсы». Приложение отображает список доступных студенту курсов. Для каждого курса выводятся название, описание, преподаватель, статус и прогресс прохождения.
Изучение учебного материала
Для изучения материала необходимо.
Открыть раздел «Курсы».
Выбрать нужный курс.
Выбрать модуль.
Открыть урок.
Ознакомиться с текстом лекции в рабочей области приложения.
Пользователь может вернуться к списку уроков и выбрать другой учебный материал.
Прохождение тестирования
Для прохождения теста необходимо.
Открыть курс.
Перейти к доступному тесту.
Запустить тестирование.
Выбрать ответы на вопросы.
Нажать кнопку завершения теста.
После отправки ответов приложение отображает результат тестирования.
Просмотр прогресса
Для просмотра прогресса необходимо перейти в раздел прогресса или открыть главный экран приложения. Пользователь может увидеть количество активных курсов, пройденных уроков и общий процент прохождения обучения.
Просмотр сертификатов
Для просмотра сертификатов необходимо открыть раздел «Сертификаты». В данном разделе отображаются сведения о полученных сертификатах: название курса, дата выдачи и уникальный номер сертификата.
Выход из приложения
Для завершения работы необходимо нажать кнопку «Выйти». После выхода приложение завершает текущую сессию и возвращает пользователя на экран авторизации.
Заключение
В ходе прохождения учебной практики было разработано настольное приложение для онлайн-школы дополнительного образования, предназначенное для предоставления студентам доступа к учебным курсам, материалам, тестированию, прогрессу обучения и сведениям о сертификатах. WPF-клиент реализован на языке C# с использованием технологии Windows Presentation Foundation и взаимодействует с серверной частью программного комплекса через REST API.
В процессе выполнения практики был проведен анализ предметной области, выявлены основные потребности пользователей онлайн-школы и обоснована необходимость создания удобного настольного клиента для работы с образовательной платформой. На основе результатов анализа была сформирована концепция WPF-приложения, обеспечивающего авторизацию студента, просмотр курсов, изучение лекционных материалов, прохождение тестов и получение результатов обучения.
На этапе разработки была спроектирована структура настольного приложения, определены основные функциональные модули, описано взаимодействие WPF-клиента с серверным API, а также разработаны формы пользовательского интерфейса. В приложении реализованы модули авторизации, профиля, курсов и уроков, тестирования, прогресса и сертификатов. Клиентская часть не обращается к базе данных напрямую, а получает и передает данные через API в формате JSON.
На заключительном этапе был подготовлен комплект технической документации, включающий пояснительную записку, техническое задание, эскизный проект, технический проект, текст программы, раздел тестирования и руководство пользователя. Также было проведено тестирование основных функций WPF-клиента, по результатам которого подтверждена корректность работы авторизации, загрузки курсов, просмотра учебных материалов, прохождения тестирования, отображения прогресса и сертификатов.
